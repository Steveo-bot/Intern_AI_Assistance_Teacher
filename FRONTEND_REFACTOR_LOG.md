# 前端模板重构日志

- 日期：2026-07-20
- 修改范围：`src`
- 修改目的：统一前端应用模板，重构全局导航，并将“发布与打印”页面升级为可编辑的作业排版工作台。
- 明确未修改：`backend`

## 1. 需求背景

本轮重构基于以下产品约束：

1. AI 教师助手只是完整系统中的一个模块，不能在全局导航中把所有功能平铺成强制步骤。
2. 全局左侧导航需要先展示模块入口，点击后再展开该模块下的子页面。
3. 子页面是可独立进入的功能，不表示用户必须完成完整线性流程。
4. “发布与打印”中的作业基本信息需要变窄，将主要空间让给作业排版与浏览区域。
5. 作业排版区域需要支持网页内直接修改内容，以及内容块自由拖动。
6. 本轮只能修改前端，不修改后端接口和数据模型。

## 2. 采用的模板方向

采用面向操作型系统的工作台模板：

```text
App Shell
├── 全局左侧导航
│   ├── 作业设计与管理
│   ├── 智能批改
│   └── 学情分析
├── 全局标题栏
└── 页面工作区
    ├── 常规业务页面
    └── 发布与打印排版工作台
        ├── 窄设置栏
        ├── 编辑工具栏
        └── A4 页面画布
```

模板没有引入新的商业 PDF SDK，也没有直接修改 PDF 二进制内容。当前方案编辑的是结构化作业内容和页面布局，后续可在此基础上接入 PDF 导出。

## 3. 代码规范依据

新建模块参考 Google TypeScript Style Guide：

- 使用 ES Module。
- 应用内部模块使用命名导出。
- 使用明确的接口定义对象结构。
- 对不应被组件修改的属性使用 `readonly`。
- 全局常量使用 `CONSTANT_CASE`。
- 类型导入使用 `import type`。
- 页面配置、布局组件和业务页面分别管理。
- 交互按钮补充 `type`、`aria-label`、`aria-expanded` 和 `aria-current`。
- 为键盘操作保留清晰的 `focus-visible` 样式。

参考资料：

- [Google TypeScript Style Guide](https://google.github.io/styleguide/tsguide.html)
- [Google Style Guides](https://google.github.io/styleguide/)
- [Material 3 Navigation Drawer](https://m3.material.io/components/navigation-drawer/overview)
- [web.dev Accessibility: Focus](https://web.dev/learn/accessibility/focus)

## 4. 新增文件

### `src/app/config/navigation.ts`

- 新增统一的 `PageId` 页面类型。
- 新增导航项、导航模块和页面元数据接口。
- 将原来平铺的页面导航整理为三个模块：
  - 作业设计与管理
  - 智能批改
  - 学情分析
- 统一管理每个页面的标题和面包屑。

### `src/app/layout/AppShell.tsx`

- 新增全局应用外壳。
- 新增统一品牌区、全局左侧导航、用户信息区和页面标题栏。
- 模块入口支持展开和折叠。
- 页面切换到其他模块时自动展开对应模块。
- 用户主动折叠当前模块后不会被状态监听立即重新展开。
- 当前模块和当前子页面分别提供视觉状态。
- 当前子页面使用 `aria-current="page"` 标记。
- “发布与打印”页面自动进入无外边距的工作台模式。

### `src/app/pages/publish/PublishWorkspace.tsx`

- 新增独立的发布与打印工作台。
- 左侧设置栏宽度收窄，用于管理：
  - 作业名称
  - 年级、学科、题目数量和总分
  - 纸张尺寸
  - 正文字号
  - 发布班级
- 主工作区新增编辑工具栏：
  - 撤销和重做入口
  - 文字和段落工具入口
  - 页面缩放
  - 打印入口
  - PDF 导出入口
- 新增固定尺寸的 A4 页面画布。
- 作业标题、副标题、学生信息和题目被建模为独立页面内容块。
- 内容块支持网页内直接编辑。
- 当前选中的内容块显示拖动手柄。
- 使用 Pointer Events 和 Pointer Capture 实现拖动状态。
- 拖动位置受到页面边界限制，避免内容块完全移出纸张。
- 提供恢复标准排版功能。
- 保留班级选择和确认发布的演示状态。

## 7. 用户可见变化

### 全局导航

修改前：

- 作业生成、审阅、发布、上传和批改等页面在导航中直接平铺。
- 视觉上容易被理解为必须依次完成的固定流程。

修改后：

- 全局导航首先展示三个模块入口。
- 点击模块后才显示子页面。
- 子页面可以独立访问，不再强调强制步骤关系。

### 发布与打印

修改前：

- 作业基本信息和排版设置占据大面积内容区。
- 作业预览区域较窄，更接近静态预览卡片。

修改后：

- 作业信息集中到窄设置栏。
- 页面主体成为接近桌面排版软件的编辑工作区。
- 用户可以直接修改画布文字。
- 用户可以选择内容块并通过手柄调整位置。
- 支持调整字号和缩放页面。

## 10. 已知限制与后续建议

1. `App.tsx` 中仍保留其余八个旧页面及部分旧版组件实现，后续应按页面拆分并使用懒加载，降低主包体积。
2. 当前页面编辑状态保存在 React 本地状态中，刷新页面后不会持久化。
3. 打印和导出 PDF 按钮目前是前端入口，尚未连接实际导出逻辑。
4. 撤销和重做按钮目前为预留状态。
5. 页面内容块当前支持移动和文字编辑，尚未实现自由缩放、跨页流动和分页碰撞处理。
6. Pointer Events 拖动逻辑已实现，但仍建议在鼠标和触控板设备上人工确认拖动手感。
7. 后续推荐将作业内容保存为 `content JSON`，将位置、尺寸和分页信息保存为 `layout JSON`，再由统一打印模块输出 PDF。
